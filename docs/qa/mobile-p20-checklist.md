# P20 移动端 H5/小程序联调清单

> 目标：在真实浏览器、HBuilderX/UniApp 环境、小程序开发者工具和生产域名下，验证移动端工具箱与 PC Web、Electron 桌面端的信息架构一致。

## 1. 前置条件

- 后端接口可访问：`/api/app/toolbox/home`、`/api/app/toolbox/study/videos`、`/api/app/message/unreadCount`、`/api/app/ai/models`。
- 管理端已有至少 1 个公开 Web 工具、1 条学习内容、1 条消息、1 个 AI 文本模型；如需多模态回归，再补图片/音频/视频模型。
- 准备 1 个 APP 用户账号，用于验证登录、收藏、消息、AI 模型和生成能力。
- 不提交真实 `.env`、模型 key、数据库密码、小程序密钥或生产 token。

## 2. 本地代码检查

在仓库根目录确认没有混入构建产物：

```bash
git status --short
git diff --check
```

移动端静态验证：

```bash
cd cool-uni-8.x
pnpm exec tsc --noEmit
```

当前仓库的 `cool-uni-8.x/vite.config.ts` 依赖 `@dcloudio/vite-plugin-uni`，但本地 `package.json` 未安装该构建器依赖，直接执行 `pnpm exec vite build --mode h5` 会失败。H5/小程序构建需在完整 UniApp/HBuilderX 环境复跑。

## 3. H5 浏览器联调

| 检查项 | 操作 | 期望 |
| --- | --- | --- |
| 首屏 | 打开工具箱 H5 首页 | Hero、搜索、Tab、工具卡片无挤压，无横向溢出 |
| 工具搜索 | 输入工具名或标签关键词 | 工具列表过滤正确；无结果时显示空状态和刷新动作 |
| 工具打开 | 点击 Web 工具 | H5 进入 `pages/toolbox/webview`，标题和 URL 正确 |
| 登录 | 点击右上角登录或我的页登录 | 登录成功后收藏、消息、AI 状态刷新 |
| 收藏 | 点亮/取消工具星标 | UI 乐观更新；失败时回滚并提示 |
| 学习 | 切换学习分类、推荐/热门 | 列表刷新，封面、标题、分类、简介显示正常 |
| AI | 登录后进入智能 Tab | 模型/模板加载；文本发送和多模态生成错误提示可读 |
| 消息 | 打开消息面板并点消息 | 未读数刷新，已读状态可回写 |
| 下拉刷新 | 在首页下拉刷新 | 当前工作区数据刷新，刷新动画结束 |
| 我的页自检 | 点击“联调自检-检查” | 工具、移动端打开、学习、AI、消息五项状态更新 |

## 4. 小程序开发者工具联调

| 检查项 | 操作 | 期望 |
| --- | --- | --- |
| 条件编译 | 打开 Web 工具 | 不进入 H5 web-view；弹出复制链接提示 |
| 链接复制 | 确认复制工具链接 | 链接写入剪贴板，toast 不遮挡主要内容 |
| 学习视频 | 点击有视频地址的学习卡片 | 小程序端复制学习地址并提示 |
| 样式兼容 | 检查首页、工具、学习、AI、我的 | 渐变、骨架屏、按钮、卡片在小程序端无明显错位 |
| 下拉刷新 | 在工具箱页下拉 | 页面触发刷新，并调用 `uni.stopPullDownRefresh()` 结束动画 |
| 登录态 | 登录后重新进入页面 | token 恢复，收藏、消息、AI 状态可继续使用 |

## 5. 生产域名联调

生产域名：`https://tool.baotounews.cn`

| 路径 | 验证 |
| --- | --- |
| `/api/app/toolbox/home` | 返回分类、推荐工具、新工具、热门工具，不暴露敏感字段 |
| `/api/app/toolbox/study/videos` | 返回学习列表，封面/视频 URL 可经 `/upload` 或外链访问 |
| `/api/app/message/unreadCount` | 未登录返回鉴权错误或 0；登录后返回未读数 |
| `/api/app/ai/models` | 登录后返回启用模型，API Key 不回显 |
| `/web` | PC Web 公开工具站可访问 |
| `/download` | 下载页可访问 |
