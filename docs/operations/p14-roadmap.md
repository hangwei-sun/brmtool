# P14 上线后运营增强路线图

## 目标

P14 聚焦上线后的运营效率，不改变 P13 发布主线。先用现有工具、消息、用户、使用统计和留言板能力形成可观察、可运营、可迭代的闭环。

## 第一批增强

- 运营数据看板：在现有使用统计基础上补充时间范围筛选、趋势图、工具转化排行、导出字段配置。
- 下载页配置化：将 macOS/Windows 下载地址、版本号、更新说明从环境变量升级为后台配置。
- 消息触达增强：支持按用户、工具使用行为、最近活跃时间筛选发送站内信。
- 留言板运营闭环：增加处理人、处理耗时、常见反馈分类和状态筛选。
- 插件体系准备：定义插件元信息、权限说明、运行入口、配置 schema 和审核状态。

## 暂缓边界

- 不在 P14 第一批引入第三方插件沙箱或任意命令执行。
- 不做短信、邮件、企业微信等外部触达。
- 不新增移动端适配目标。
- 不替代 P13 的签名、公证、更新链路验证任务。

## 验收口径

- 后台运营人员能看到上线后的核心使用数据，并导出用户与工具使用明细。
- 下载页下载地址和版本展示可由后台维护，不需要重新构建管理端。
- 消息通知能按基础人群做定向发布，并能看到已读情况。
- 留言板建议能按状态和分类跟进，形成可追踪处理闭环。
